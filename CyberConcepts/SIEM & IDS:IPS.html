AS Tools

Category Function Tools Details
SIEM
(Security Information and Event Management)		 Collects, centralizes, and analyzes data from various sources Splunk Enterprise / Splunk Cloud Data from firewalls (Palo Alto, Check Point, Fortinet), IDS, endpoint protection, network devices, and cloud environments.
Includes Machine Learning Toolkit (MLTK).
Splunk Phantom: SOAR tool for automating incident response workflows.
Microsoft Sentinel Cloud-based SIEM and SOAR solution (Azure).
Collects data from firewalls, network devices, Microsoft Defender, Azure AD, VMware.
Stores data in Log Analytics using Kusto Query Language (KQL).
Allows playbook creation (e.g., system isolation, IP/user blocking).
Reports are generated and shared with the IT support team.
IDS/IPS
(Intrusion Detection / Prevention Systems)		 Monitors network traffic and detects potential intrusions or abnormal behavior Snort Supports protocol analysis (TCP, UDP, ICMP, HTTP, DNS, etc.).
Can output alerts to Syslog, SNMP traps, and MySQL.
Suricata More advanced than Snort.
Supports HTTP/2, IPv6, TLS 1.3.
Outputs alerts in JSON format.

IDS Types

IDS Type Description Example
Signature-Based IDS Detects known attack patterns by comparing traffic to a database of signatures. Snort, Suricata
Anomaly-Based IDS Detects deviations from established baselines, identifying unusual behavior. Bro IDS, OSSEC
Stateful Protocol Analysis IDS Monitors the state and behavior of protocols to detect deviations from normal state transitions. IBM Proventia, McAfee Network Security Platform
Hybrid IDS Combines the features of signature-based and anomaly-based detection systems. Intrusion Detection System that uses both traffic pattern analysis and predefined signatures.
Host-Based IDS Monitors activity on a specific device or host, detecting abnormal activity. OSSEC, Tripwire
Network-Based IDS Monitors network traffic to detect potential threats or malicious activity. Snort, Suricata
