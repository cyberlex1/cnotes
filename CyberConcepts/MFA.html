Types of Authentication Factors

Authentication Factor Description Example
Something You Know (Knowledge Factor) This is typically a password, PIN, or passphrase. A user enters their password during login.
Something You Have (Possession Factor) This refers to a physical object that the user has, such as a mobile phone, hardware token, or smart card. A user receives a one-time passcode (OTP) on their phone or uses an authentication app like Google Authenticator to generate a code.
Something You Are (Inherence Factor) This factor involves biometric verification, such as fingerprints, face recognition, or iris scanning. A user uses their fingerprint to unlock a device or verify their identity.
Somewhere You Are (Location Factor) This is based on the user’s geographical location, detected through IP addresses, GPS, or Wi-Fi networks. A user can only log in when accessing from a specific geographic location, like a corporate network.

How MFA Works Step-by-Step

Step Action Authentication Factor
1 User Logs In with Username and Password First Factor: Something You Know (Password)
2 Second Authentication Factor is Requested Second Factor Options:
  • One-time passcode (OTP) sent to phone or email (Something You Have)
  • Biometric scan (fingerprint or face scan) (Something You Are)
  • Smart card or security key (Something You Have)
3 User Provides the Second Factor Second Factor: User enters OTP, performs biometric scan, or uses security key
4 Access Granted Both factors are verified, and the user gains access to the system

Common Forms of MFA

Form Description
SMS-based OTP A one-time passcode sent via text message
Authenticator Apps Apps like Google Authenticator or Microsoft Authenticator generate time-based one-time passcodes
Hardware Tokens Physical devices that generate or display one-time passcodes
Biometric Scanning Fingerprint, facial recognition, or iris scanning
Push Notifications A notification sent to the user’s mobile device for approval or denial of the login attempt

Benefits of MFA

Benefit Description
Increased Security Even if an attacker knows the password, they still need access to the second factor (e.g., phone, fingerprint)
Protection Against Phishing Even if a password is stolen, MFA adds another layer of defense
Compliance Many industries and regulations require MFA for sensitive data (e.g., financial services, healthcare)

Common MFA Attacks

Attack Description
SIM Swapping Attackers trick mobile providers into transferring a victim’s phone number to a new SIM card, intercepting OTPs
Phishing Attackers create fake login pages to steal passwords and OTPs from users
MFA Fatigue Overloading a user with repeated authentication requests to prompt them to approve a malicious login attempt
