Penetration Testing on LLMs

  • Prompt Injection: Manipulating input prompts to make AI models generate unintended or harmful outputs.
  • Indirect Prompt Injection: Influencing the context around an AI model's response to achieve malicious results without directly altering the prompt.
  • Training Data Poisoning: Inserting malicious data into the training set to corrupt the AI model's behavior.
  • Hallucination: When AI generates plausible but false or incorrect information.

Vulnerability Assessment & Management

was responsible for managing the Vulnerability Assessment & Management process using DefectDojo as the central platform. The project aimed to consolidate and manage vulnerabilities from multiple security testing tools (SAST, DAST, IAST, SCA across the development lifecycle).

Vulnerability Assessment Lifecycle
  • Preparation & Planning: Define scope, identify critical assets, and select appropriate Assessment Tools (AS Tools).
  • Asset Discovery: Identify and inventory all systems and applications in scope.
  • Vulnerability Scanning: Use tools like SAST, DAST, IAST, and SCA to detect vulnerabilities.
  • Vulnerability Analysis: Assess severity using CVSS score and eliminate false positives.
  • Reporting: Document and communicate findings to stakeholders.
  • Remediation: Apply fixes and patches to address identified issues.
  • Retesting: Validate that remediations were successful and no new issues have been introduced.
  • Continuous Monitoring: Automate scans and use SIEM tools to maintain security posture over time.
