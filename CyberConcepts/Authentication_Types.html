Authentication Type Description Protocol Type Encryption Common Attack (Educational)
Password-based User provides a password to authenticate Varies (HTTP, SSH, etc.) Usually hashed (MD5, SHA-1, bcrypt, etc.) Password spraying, brute-force, phishing
Multi-Factor Authentication (MFA) Combines two or more authentication factors Protocol add-on (TOTP, Push, SMS) TOTP encrypted (HMAC, SHA-1) Social engineering, SIM swapping, MFA fatigue attacks
Biometric Uses fingerprint, face, or iris recognition Custom/Hardware specific Device-level encryption Sensor spoofing, replay attacks
Token-based Generates a token for session-based authentication OAuth, JWT JWT (Base64 + Signature), TLS Token hijacking, XSS to steal tokens
Certificate-based Uses X.509 certificates for identity validation SSL/TLS, PKI Public-key encryption (RSA, ECC) Man-in-the-middle (if certs are weak/expired)
Smart Card Physical card with embedded chip stores credentials PKI, PIV Asymmetric (RSA) + PIN Card cloning, stolen card with PIN
Single Sign-On (SSO) One login grants access to multiple services SAML, OAuth, OIDC Depends on backend (JWT, XML Sig) SSO token abuse, misconfigured identity providers
Kerberos Uses tickets for mutual authentication Kerberos (UDP/TCP 88) Symmetric (AES, RC4) Kerberoasting, AS-REP Roasting, Pass-the-Ticket
NTLM Microsoft challenge-response auth protocol NTLMv1/v2 MD4/NT hash Pass-the-Hash, relay attacks
RADIUS Centralized AAA protocol (Access, Auth, Accounting) RADIUS Depends (can be PAP, CHAP, EAP) Credential replay, weak shared secrets
LDAP Protocol for accessing directory services LDAP / LDAPS None (LDAP) / TLS (LDAPS) LDAP injection, cleartext creds sniffing (LDAP)
SAML XML-based SSO auth for enterprise apps SAML (over HTTP) XML Signatures, TLS Signature wrapping, replay attacks
OAuth Delegated access without password sharing OAuth 2.0 Token-based, TLS Token reuse, redirect URI manipulation
OpenID Connect Identity layer on top of OAuth 2.0 OIDC (OAuth + JWT) JWT + TLS ID token theft, insecure token storage
