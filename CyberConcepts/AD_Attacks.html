Attack Access Level Tool Description
Kerbrute Enumeration No domain access Kerbrute Enumerates valid AD users via Kerberos pre-auth abuse.
Harvesting User access Rubeus Collects Kerberos tickets for later use (e.g., Pass-the-Ticket).
Password Spraying User access Rubeus Tries one password across many users to find weak creds.

AS-REP Roasting Any user Rubeus Cracks encrypted TGTs of users with no pre-auth enabled.
Pass-the-Ticket Domain Admin (if TGT from DA) Mimikatz Uses dumped TGTs to impersonate users and escalate privileges.
Golden Ticket Maintain access (full) Mimikatz Create forged TGTs using krbtgt hash—full domain access.
Silver Ticket Maintain access (service) Mimikatz Forged ticket for specific service—stealthier than Golden Ticket.
Skeleton Key Maintain access (domain-wide) Mimikatz Backdoor master key allowing domain-wide access via Kerberos.
  • How does Responder work? Responder is used in internal network attacks to capture NTLM hashes by poisoning name resolution protocols like LLMNR, NBT-NS, and MDNS.
  • How does NTLM Relaying work? NTLM relaying is a Man-in-the-Middle (MitM) attack where the attacker intercepts an NTLM authentication request and forwards it to another service to gain access without cracking the hash.
  • Attack path to compromise Domain Admin:
    • Enumerate valid Active Directory (AD) users with kerbrute.
    • Gain initial access to a Domain User through password spraying.
    • Query the domain controller to identify service accounts with Domain Admin privileges and perform kerberoasting to get TGS tickets.
    • Crack the TGS ticket using hashcat to retrieve the cleartext password.
    • Login with Domain Admin credentials using Runas or PsExec.
