SIEM vs IDS/IPS vs IDR
SIEM is more about monitoring and correlating data from various sources and providing insights into threats and incidents.
IPS/IDS are focused on detecting and preventing threats in real-time, mainly at the network level.
IDR combines incident detection and response, focusing on identifying and mitigating incidents as they occur, often with automated remediation.
These systems are complementary in a security infrastructure. For example, a SIEM could be used to collect and analyze logs, IDS might detect suspicious activities, and IDR systems might provide the tools to quickly respond to and mitigate any detected incidents.

AS Tools
🔐 SAST (Static Application Security Testing)
  • Scans source code for vulnerabilities before execution.
  • Detects issues like insecure coding, buffer overflows, and SQL injection.
  • Integrated early in the development phase of the SDLC.
🔐 DAST (Dynamic Application Security Testing)
  • Tests running applications to identify vulnerabilities.
  • Simulates external attacks like authentication issues and session management flaws.
  • Used in the testing or staging phase of the SDLC.
🔐 IAST (Interactive Application Security Testing)
  • Combines SAST and DAST.
  • Provides real-time insights into source code vulnerabilities while the application is running.
  • Used throughout the SDLC, especially during QA testing.
🔐 SCA (Software Composition Analysis)
  • Analyzes open-source libraries and dependencies for known vulnerabilities.
  • Ensures third-party components are secure and compliant with licensing requirements.
  • Integrated into the SDLC to manage risks from outdated or vulnerable libraries.
